# LumaClaw Reset Draft Vision

The reset target is the smallest credible LumaClaw mainline that can stand on its own.

## Vision for the reset

- preserve `LumaClaw` as the default identity
- preserve SoulBond as the core differentiator
- preserve a small project-truth layer so the repo stays explainable
- reject inherited complexity that is not needed for the phase-one reset

## Reset thesis

The new mainline should inherit project truth and core differentiation, not the full weight of migration-era residue.

That means the reset should prefer:

- one default product identity
- one default entrypoint
- a narrow dependency set
- a small, reviewable carry list

## Not part of this draft

- broad upstream compatibility
- broad UI migration
- apps, extensions, skills, or workflow breadth
- top-level file replacement before acceptance
